## 引入客户端

```xml
<!-- 1 -->
<dependency>
            <groupId>com.powercloud</groupId>
            <artifactId>power-comm</artifactId>
            <version>3.1.1</version>
        </dependency>

<!--或者直接引入安全认证客户端，建议属于第一种-->

<!-- 2 -->
<dependency>
            <groupId>com.powercloud</groupId>
            <artifactId>power-auth</artifactId>
            <version>3.0.3</version>
        </dependency>
```

## 添加认证安全组件相关参数配置

```properties
#=====[start]所需相关配置=========#单点登录模式hsa-sso（正式）hsa-mock（测试，默认返回mock用户）
#认证模式power-sso（正式）power-mock（测试，默认返回mock 用户）
power.security.type=power-sso
#客户端允许访问的白名单，比如:/login/**,/resource/**,/error,/**/**.shtml,/home/index.action
#框架默认配置：/login/**,/resource/**,/error,/**/**.shtml,/home/index.action,/doc.html,/webjars/**,/v2/api-docs,/swagger-resources/**,/remoting.call,/powercloud/**";
power.security.client.permits
#是否限制接口权限，开启认证模式之后，可以设置api-mock，不限制接口权限
power.security.api.type=api-sso
#允许调用的ip，不配置允许全部
power.security.referer.permits.ip
#mock调式的模式，支持平台管理员用户、开发商团队用户两种调式；调式平台管理员权限，配置为：mock-user
power.security.mock.mode:mock-user-acc
#=========[end]============================================
```

## 获取认证相关信息
``` 
//获取当前登录用户信息
PowerContextHolder.getContext().getPowerPortalUser();
//判断当前用户是否绑定了开发团队，绑定了开发团队只属于操作当前开发团队
PowerContextHolder.getContext().getPowerPortalUser().isBindAcc();
//获取当前acc列表，如果当前用户绑定了开发团队，那么列表只包含绑定的开发团队信息
PowerContextHolder.getContext().getCloudAccList();
```
